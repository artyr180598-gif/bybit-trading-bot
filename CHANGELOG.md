## v6.1 — 5 Апреля 2026 (текущая версия)

### 🔧 Исправление: строгие фильтры блокировали сигналы в медвежьем рынке

**Диагностика показала:** при RSI=20-28 (крипта падает) три фильтра одновременно
блокировали ВСЕ сигналы — бот простаивал всю ночь.

| Параметр | Было | Стало | Причина изменения |
|----------|------|-------|-------------------|
| `EMA_SEP_MIN_PCT` | 0.12% | **0.06%** | EMA21/50 сходятся до 0.04-0.10% при коррекции |
| `SRSI_OS` (SHORT) | 15 | **5** | StochRSI=0-12 в медвежьем → все SHORTs блокированы |
| `RSI_SHORT_MIN` | 28 | **22** | RSI падает до 20-28 в коррекции, надо торговать |
| `RSI_SHORT_MAX` | 58 | **62** | Чуть шире зона SHORT |
| `ADX_MIN` | 20 | **18** | Умеренные тренды тоже берём |
| `VOL_RATIO_MIN` | 0.75 | **0.60** | 60% объёма достаточно |

**Важно:** В экстремальных условиях (RSI < 22 на всех монетах) бот всё равно ждёт.
Продавать при RSI=20-22 = продавать дно капитуляции. Это **правильно**.

## v6.0 — 4 Апреля 2026

### 🚀 Главное: Новая стратегия — TP×6 вместо TP×1.5

**Найден root cause низкой доходности исходного бота:**
```
Исходный: TP1 = SL = 1.5×ATR  →  R:R = 1:1
При WR=55% и комиссии 0.11%/сделка → EV ≈ 0 → доходность ~2-27%/год
```

**Решение — увеличить R:R до 4:1:**
```python
# Было:
ATR_TP_MULT  = 1.5   # TP1
ATR_TP2_MULT = 3.0   # TP2
RISK_PCT     = 2.5%

# Стало (v6):
ATR_TP_MULT  = 3.0   # TP1 — R:R 2:1
ATR_TP2_MULT = 6.0   # TP2 — R:R 4:1  ← ключевое изменение
RISK_PCT     = 3.0%
```

**Математика:** При R:R=4:1 достаточно WR=20% для прибыли.
WR=28% (реальный) → EV = 0.28×4 - 0.72×1 = **+0.40 за единицу риска**.

### ✅ Новые фильтры

**1. EMA200 — тройной тренд-фильтр (самое важное)**
```python
EMA_XSLOW = 200  # добавлен в calc_indicators()
# Условие LONG:  EMA21 > EMA50 > EMA200
# Условие SHORT: EMA21 < EMA50 < EMA200
```
*Без EMA200 бот входил в LONG посреди нисходящего мега-тренда.*

**2. StochRSI — фильтр перекупленности** ⚠️ КРИТИЧЕСКИ ВАЖЕН
```python
SRSI_OB = 82  # не LONG если перекуплен (StochRSI > 82)
SRSI_OS = 5   # не SHORT если экстремально перепродан (StochRSI < 5)
```
*Бэктест: БЕЗ StochRSI = -1.7%/год. С ним = +40%/год. Никогда не убирать.*

**3. Volume — подтверждение объёмом**
```python
VOL_RATIO_MIN = 0.60  # объём ≥ 60% от 20-периодной MA
```

### ❌ Убрали: 2-барное подтверждение
Задерживало вход на 1H свечу, теряли тренды. Заменено на StochRSI + Volume.

## ⚠️ Технические особенности и ограничения

### Bybit API заблокирован с Replit IP
**Решение:** Везде используем OKX API как источник данных
```python
# fetch_klines() → при ошибке Bybit → автоматически fetch_klines_okx()
# Свечи: https://www.okx.com/api/v5/market/history-candles (публичный)
# Цена:  https://www.okx.com/api/v5/market/ticker (публичный)
```

### Высокая просадка (DD ~70%)
Это следствие R:R=4:1 с Risk=3%: редкие большие wins, частые малые losses.
**Защита:**
- `GLOBAL_DD = 30%` — circuit-breaker, останавливает торговлю при -30% от пика
- `DAY_LOSS_PCT = 15%` — дневной лимит потерь

### Бот молчит при RSI < 22 — это нормально
В экстремальных распродажах (5-8 раз/год) бот ждёт стабилизации.
Проверить текущее состояние: `python check_signals.py` (если есть в репо)

### Торговля начинается когда RSI возвращается в 35-50
После паники рынок восстанавливается → EMA расходятся → ADX растёт → сигналы появляются.
